# Binary Search & Linked Lists

## Binary search
Not just "find x in sorted array" — it's *"the answer space is monotonic, so I can halve it."*

```python
def search(lo, hi, ok):           # generic template
    while lo < hi:
        mid = (lo + hi) // 2
        if ok(mid): hi = mid      # answer is mid or left
        else: lo = mid + 1
    return lo
```

Levels of difficulty: classic search → Search in Rotated Sorted Array (one half is always sorted — find it) → Find Minimum in Rotated Array → Koko Eating Bananas (binary search **on the answer**, the pattern interviewers love at senior level).

Off-by-one discipline: pick `[lo, hi)` or `[lo, hi]` and never mix conventions mid-problem.

## Linked lists
Less common at big tech now but still appears in screens. Master four mechanical skills:
1. **Reversal** (three-pointer dance: `prev, cur, nxt`).
2. **Slow/fast pointers** — middle of list, cycle detection (Floyd), cycle start.
3. **Dummy head** — eliminates head-edge-case bugs in merge/remove problems.
4. **Merge two sorted lists** — the building block of Merge K (which adds a heap).

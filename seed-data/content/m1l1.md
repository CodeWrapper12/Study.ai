# Setup & the Grind System (Python for C# Devs)

**Why this lesson exists:** every big-tech loop (Microsoft, Amazon, Careem, Talabat) opens with timed coding rounds. Python halves your keystrokes versus C#, which buys you thinking time. You already know Python casually — this lesson makes it your interview weapon.

## Python ↔ C# translation table

| C# | Python |
|---|---|
| `Dictionary<K,V>` | `dict` / `collections.defaultdict` |
| `HashSet<T>` | `set` |
| `List<T>` | `list` |
| `Queue<T>` | `collections.deque` (use `popleft()`) |
| `PriorityQueue` | `heapq` (min-heap; negate for max) |
| `StringBuilder` | `''.join(list_of_strings)` |
| LINQ `OrderBy` | `sorted(xs, key=lambda x: ...)` |

## The grind system

- **Daily, not weekends:** 1–2 problems per day (60–90 min) beats 8-hour Saturday marathons. Consistency builds pattern recognition; cramming builds fatigue.
- **Pattern-first:** follow a curated list (NeetCode 150 ordering) so problems cluster by technique.
- **Spaced repetition:** keep a "failed" list. Re-solve every failed problem after 3 days, then 7, then 21 — from a blank editor.
- **Time-box:** 25 minutes stuck → read the approach (not the code), close it, implement yourself.

## Lab
1. Install Python 3.12+, set up a `grind/` repo with one folder per pattern.
2. Solve *Two Sum* and *Contains Duplicate* in Python today.
3. Create your tracking sheet: problem, date, result, next-review date.

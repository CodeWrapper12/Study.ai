# LAB: Add Production-Grade RAG to a FastTrack-Style Assistant

The M6 capstone: extend RelayFlow (or a standalone repo) with a documented, evaluated RAG feature. Target: 2–3 evenings. This becomes your "AI-enabled architect" proof.

## Build: "Ops Copilot"
An assistant that answers operational questions over a small corpus you assemble: 10–15 markdown docs (fake SOPs, partner SLA excerpts, escalation runbooks — write them with an LLM's help, they just need realistic structure).

1. **Ingestion pipeline** (console app or background service): chunk by headings with overlap, embed (OpenAI `text-embedding-3-small` or a local model), store in **pgvector** — PostgreSQL again, deliberately: one less moving part, and "pgvector vs dedicated vector DB" is itself an interview talking point (start simple, migrate when scale demands).
2. **Hybrid retrieval:** pgvector cosine + PostgreSQL full-text search, fused with RRF in C#. Filter by an `access_level` metadata column based on the caller's role — RBAC meets RAG.
3. **Answer generation** with mandatory citations: response schema includes `sources: [chunkIds]`; render them. No citation → regenerate or refuse.
4. **Evals from day one:** 25-case eval set + runner (lesson 3's method). README publishes the pass rate and one example of a failure the evals caught — that honesty reads as seniority.
5. **Stretch:** wire it as a 10th Semantic Kernel plugin pattern — `SearchOpsDocs` as a tool the assistant calls, demonstrating RAG-as-tool rather than RAG-as-pipeline.

## Definition of done
`docker compose up` → ask "what's the escalation path when a partner's lounge is overbooked?" → grounded, cited answer. README: architecture diagram, chunking rationale, hybrid-search explanation, eval results table.

## The interview paragraph you've earned
*"I added RAG to an operational assistant: heading-aware chunking, hybrid pgvector+FTS retrieval with RRF, role-filtered access, enforced citations, and a 25-case eval suite in CI. Retrieval quality, not the model, was where all the real work lived."*

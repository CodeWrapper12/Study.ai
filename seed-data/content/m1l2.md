# Arrays, Hashing & Two Pointers

**Interview frequency: highest.** Roughly a third of all screening questions are array/string problems wearing costumes.

## Hashing pattern
When you hear *"find a pair / count occurrences / check existence"* — reach for a `dict` or `set` to trade memory for an O(n) pass.

```python
def two_sum(nums, target):
    seen = {}                      # value -> index
    for i, n in enumerate(nums):
        if target - n in seen:
            return [seen[target - n], i]
        seen[n] = i
```

Key questions in this family: Two Sum, Group Anagrams (frequency-count key), Top K Frequent, Longest Consecutive Sequence (set + only start counting at sequence heads).

## Two pointers
Works when the input is **sorted** or you converge from both ends: Valid Palindrome, Two Sum II, 3Sum (sort + fix one + two-pointer the rest), Container With Most Water (move the shorter wall).

## How to talk while solving
1. Restate the problem + clarify constraints (sorted? duplicates? size?).
2. State brute force and its complexity *out loud*.
3. Name the pattern ("I'll trade space for time with a hash map").
4. Code, then dry-run one example and one edge case.

# Evals & Guardrails for Production LLM Features

The question that exposes demo-builders: *"How do you know your AI feature still works after you change the prompt?"* If the answer is "we try a few questions," it's a demo. Evals are the testing discipline for the LLM era — and your strongest AI interview differentiator.

## Evals = unit tests for model behavior
- Build a **versioned eval set**: 50–200 real cases of input → expected behavior. Sources: production logs (gold), support tickets, your own red-teaming.
- Run on every prompt/model/retrieval change, in CI ideally. Track pass-rate over time like you track test coverage.
- Grade by type: exact-match where possible (did NL→SQL produce the right rows?), **LLM-as-judge** for open-ended outputs (a second model scores against a rubric), with periodic human calibration so the judge doesn't drift.
- The NL→SQL system you built is perfectly evaluable: question → expected result-set hash. Start there; it's a CV bullet within a week.

## Guardrails — layered, never single
1. **Input:** intent classification (your router already does this), prompt-injection screening, off-topic deflection.
2. **Constrained generation:** the model chooses from a semantic layer (your ViewCatalog!) rather than free-generating SQL; structured outputs with schema validation.
3. **Output validation:** SQL validator/allow-lists (you built one), PII scrubbing, citation enforcement, toxicity screening where user-facing.
4. **Execution sandbox:** read-only DB connections, row limits, timeouts — blast-radius thinking again.
5. **Human escalation:** low-confidence → hand off, with the confidence signal logged.

You have personally built items 1, 2, 3 and 4 in production. This lesson's job is mostly giving you the *vocabulary* to present it: you've been doing layered guardrails and just called it common sense.

## Lab
Build a 30-case eval set for your NL→SQL reporting feature (or RelayFlow's assistant): 20 happy path, 5 ambiguous, 5 adversarial (injection attempts, out-of-scope). Script the runner; record the baseline pass rate. Congratulations — you now out-interview 90% of "AI experience" candidates.

## 1. Trees: recursion with a contract

Every tree DFS is the same three-line contract:

1. **Base case:** what is the answer for `None`?
2. **Recurse:** trust the function to answer for left and right. (*Trust* is the skill — candidates who re-verify the recursion mentally on every call drown.)
3. **Combine:** build this node's answer from the children's answers.

```python
def max_depth(root) -> int:
    if not root: return 0                       # contract for the empty tree
    return 1 + max(max_depth(root.left), max_depth(root.right))
```

Invert Binary Tree, Same Tree, Subtree of Another Tree — all this contract with different combine steps. When stuck on any tree problem, literally ask aloud: *"if I had the answers for both subtrees, how would I combine them?"* That question is the skeleton key, and saying it shows the interviewer your process.

### When the answer isn't the return value: Diameter
The diameter (longest node-to-node path) might not pass through the root — so the recursion *returns height* while *recording* diameter as a side effect:

```python
def diameter(root) -> int:
    best = 0
    def height(node) -> int:
        nonlocal best
        if not node: return 0
        lh, rh = height(node.left), height(node.right)
        best = max(best, lh + rh)               # path through this node
        return 1 + max(lh, rh)
    height(root)
    return best
```

This **return-one-thing, track-another** shape recurs in Balanced Binary Tree, Max Path Sum, and many "best X in tree" problems. Recognize it as its own mini-pattern.

### BST: the bounds-passing trap
Validate BST is the most failed easy-looking question in screens. The wrong answer checks each node against its children. The right answer passes **inherited bounds** down:

```python
def is_valid_bst(root) -> bool:
    def valid(node, lo, hi) -> bool:
        if not node: return True
        if not (lo < node.val < hi): return False
        return valid(node.left, lo, node.val) and valid(node.right, node.val, hi)
    return valid(root, float('-inf'), float('inf'))
```

Why parent-child checking fails: a grandchild can violate a *grandparent's* constraint while satisfying its parent's. Sketch the counterexample (root 5, right child 7, right-left grandchild 4) once on paper; you'll never forget it. Alternative senior answer: in-order traversal of a BST yields sorted values — validate by checking the sequence. Offering both, with trade-offs, is an instant level signal.

### BFS: when level structure is the question
```python
from collections import deque

def level_order(root) -> list[list[int]]:
    if not root: return []
    res, q = [], deque([root])
    while q:
        level = []
        for _ in range(len(q)):                 # freeze level size BEFORE the loop
            node = q.popleft()
            level.append(node.val)
            if node.left: q.append(node.left)
            if node.right: q.append(node.right)
        res.append(level)
    return res
```
The `for _ in range(len(q))` snapshot is the load-bearing trick — it's what separates levels. Right Side View = last element per level; Zigzag = alternate reversal. One template, many problems.

## 2. Graphs: first recognize, then traverse

The traversals are easy; the skill is **seeing the graph**. Graphs hide as: grids (cells = nodes, adjacency = edges), prerequisite lists (course → course edges), word relationships (Word Ladder: words differing by one letter), processes/states (your saga state machines are literally directed graphs — say so when asked where graphs appear in real systems).

### The universal toolkit
```python
from collections import deque, defaultdict

def bfs(start, graph):
    seen, q = {start}, deque([start])
    while q:
        node = q.popleft()
        for nxt in graph[node]:
            if nxt not in seen:
                seen.add(nxt)                   # mark on ENQUEUE, not on dequeue
                q.append(nxt)
```
Marking on enqueue prevents the same node entering the queue twice — a correctness *and* memory bug if done late. For DFS, the recursive form is cleanest; for inputs ≥ ~10⁵ nodes, mention Python's ~1000-frame recursion limit and offer the explicit-stack rewrite. That single sentence ("at this scale I'd convert to an iterative DFS to avoid stack overflow") is a senior tell.

**Choosing:** BFS when *distance/levels/shortest-in-unweighted* matters. DFS when *exhaustion/connectivity/path-existence* matters and memory favors depth (BFS queues can hold an entire level — exponential in branching factor for wide graphs).

### Grid flood-fill: Number of Islands
```python
def num_islands(grid: list[list[str]]) -> int:
    rows, cols = len(grid), len(grid[0])
    def sink(r, c):
        if not (0 <= r < rows and 0 <= c < cols) or grid[r][c] != '1':
            return
        grid[r][c] = '0'                        # mark visited by mutation
        for dr, dc in ((1,0),(-1,0),(0,1),(0,-1)):
            sink(r + dr, c + dc)
    count = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                count += 1
                sink(r, c)
    return count
```
Variants: Max Area of Island (sink returns size), Rotting Oranges (multi-source BFS — seed the queue with *all* rotten cells at distance 0, a genuinely important variation that models "infection from many origins"), Pacific Atlantic (reverse-flow trick: BFS from the oceans inward).

### Topological sort: dependency resolution
**Course Schedule** = "is this dependency graph acyclic?" — which you've solved professionally every time a DI container or build system resolved ordering. Kahn's algorithm, the interview-friendly version:

```python
def can_finish(num: int, prereqs: list[list[int]]) -> bool:
    graph = defaultdict(list)
    indegree = [0] * num
    for course, pre in prereqs:
        graph[pre].append(course)
        indegree[course] += 1
    q = deque(i for i in range(num) if indegree[i] == 0)
    taken = 0
    while q:
        node = q.popleft()
        taken += 1
        for nxt in graph[node]:
            indegree[nxt] -= 1
            if indegree[nxt] == 0:
                q.append(nxt)
    return taken == num                          # leftovers ⇒ cycle
```
Narrate it as: "repeatedly take anything with no unmet prerequisites; if something's left, there's a cycle." Course Schedule II returns the take-order — the same code collecting nodes. This *is* NuGet/npm dependency resolution; make that connection out loud.

### One sentence each (screens rarely require coding these)
**Dijkstra:** BFS with a min-heap when edges have weights. **Union-Find:** near-O(1) incremental connectivity ("are these connected?" under additions) — used by Number of Connected Components and Redundant Connection; learn the 15-line implementation once. **Word Ladder:** BFS where words are nodes — the lesson is graph *construction* from non-graph input.

## 3. How this breaks in production

Cycle detection guards real systems: circular DI registrations, circular workflow definitions, circular foreign keys in migration ordering. Multi-source BFS is cache-warming from N seed points and infection/outage blast-radius modeling. Topological sort runs your CI pipeline's job ordering. You have lived these; the interview asks you to name them.

## 4. Interview lens

- Trees: state the contract ("base case, recurse, combine") before coding; verify with a 3-node tree, never a 15-node one.
- Graphs: before any traversal, say what nodes and edges *are* — for grids, literally "cells are nodes, 4-directional adjacency are edges."
- Always address visited-tracking and cycle-safety explicitly; forgetting `seen` on a cyclic graph is an infinite loop in front of an audience.
- Complexity language: O(V + E) for traversals; for grids V = R·C and E ≤ 4V, so "O(rows × cols)".

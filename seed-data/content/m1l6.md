# Heaps, Backtracking & DP Patterns

## Heaps (`heapq`)
Min-heap only in Python — negate values for max-heap. Reach for it on *"K-th / top K / closest K"* and streaming-median problems: Kth Largest Element, Top K Frequent, K Closest Points, Find Median from Data Stream (two heaps).

## Backtracking
Controlled brute force: choose → explore → un-choose.

```python
def subsets(nums):
    res, path = [], []
    def bt(i):
        if i == len(nums):
            res.append(path[:]); return
        bt(i + 1)                      # exclude
        path.append(nums[i]); bt(i + 1); path.pop()  # include
    bt(0); return res
```

Core set: Subsets, Permutations, Combination Sum, Word Search. Always state the time complexity honestly (exponential) and the pruning that keeps it feasible.

## Dynamic programming — pattern recognition, not memorization
Ask: *can I define `dp[i]` as the answer for a prefix, and write a recurrence?*

- **1-D:** Climbing Stairs, House Robber (`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`), Coin Change.
- **2-D:** Unique Paths, Longest Common Subsequence.
- Start top-down with `@functools.cache`, convert to bottom-up if asked.

For senior roles you rarely get hard DP in screens — solid mediums + clear articulation beat memorized hards.

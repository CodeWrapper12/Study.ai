# LAB: Terraform an AKS + PostgreSQL Environment

The capstone for M5 — and the repo that proves "IaC" on your CV isn't a keyword. Target: one weekend. Use the Azure free tier / smallest SKUs; destroy when done.

## Build
A `relayflow-infra` repo deploying the M3/M4 application stack's home:

1. **Module structure:**
```
modules/
  network/        # vnet, subnets, NSGs
  aks/            # cluster, system+user node pools, managed identity
  postgres/       # flexible server, private endpoint, databases
  observability/  # log analytics workspace, app insights
envs/
  dev/            # main.tf composing modules + dev.tfvars
```
2. **Non-negotiables:** remote state in Azure Storage with locking; managed identity for AKS→ACR and AKS→Key Vault (zero passwords in app config); PostgreSQL behind a private endpoint with public access disabled; tags (`owner`, `env`, `costcenter`) enforced via a `locals` block.
3. **Deploy RelayFlow onto it:** push images to ACR, apply your M4 Kubernetes manifests, confirm the booking flow works end-to-end in the cloud.
4. **The money step:** screenshot the cost analysis after 48 hours, then `terraform destroy` and prove the bill stops. Document both in the README — *"I can stand up and tear down the entire environment in 20 minutes"* is the sentence hiring managers remember.

## Stretch
A GitHub Actions workflow running `terraform plan` on PR with the plan posted as a comment, and `apply` on merge to main with OIDC federation (no stored credentials). That's the modern platform-engineering loop in one workflow file.

## Definition of done
Fresh clone → `terraform apply` → working app URL → `terraform destroy` → zero residual cost. README documents the architecture diagram, identity model, and the dev/prod difference being only tfvars.
